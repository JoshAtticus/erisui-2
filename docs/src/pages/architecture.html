Single Page Application

ErisUI applications are built as Single Page Applications (SPAs). This means that only one HTML file (`index.html`) is loaded by the browser, and JavaScript handles all content updates as the user navigates.

The App Shell

The `index.html` file serves as the "App Shell". It contains the persistent UI elements like the navigation sidebar and the main content container.

Why manual? Unlike meta-frameworks (Next.js, Nuxt) that abstract this structure away, ErisUI gives you full control over your `index.html`. This approach allows you to easily include external scripts, modify meta tags, and understand exactly how your app is constructed without build-step magic.

<body> <div id="app"> <eui-app-nav></eui-app-nav> <div class="app-wrapper"> <!-- Pages are rendered here --> <div id="main"></div> </div> </div> </body> Entry Point

src/main.js is the JavaScript entry point. It is responsible for:

  • Importing global styles.
  • Registering custom web components.
  • Configuring the router.
  • Setting up the navigation menu.
Web Components

ErisUI uses standard Web Components. This means they are framework-agnostic and work in any modern browser. Custom elements are defined in the src/components directory and must be imported in main.js to be used.
